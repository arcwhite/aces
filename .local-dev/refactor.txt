### 5. campaigns.ex - update_deployment_results/2 (lines 314-325)

**Problem**: Pattern matches `{:ok, _}` which will crash on errors.

**Current**:
```elixir
Enum.map(deployments, fn deployment ->
  {deployment_id, result_attrs} = ...
  {:ok, updated} =
    deployment
    |> Deployment.post_battle_changeset(result_attrs)
    |> Repo.update()
  updated
end)
```

**Improvement**: Use `Ecto.Multi` for transaction safety:
```elixir
def update_deployment_results(deployments, results) do
  deployments
  |> Enum.reduce(Ecto.Multi.new(), fn deployment, multi ->
    {deployment_id, result_attrs} = find_result(deployment.id, results)
    changeset = Deployment.post_battle_changeset(deployment, result_attrs)
    Ecto.Multi.update(multi, {:deployment, deployment.id}, changeset)
  end)
  |> Repo.transaction()
end
```

### 6. sortie_summary.ex - enrich_allocations/1 (lines 48-71)

**Problem**: N+N performance issue - `Enum.find` inside map creates O(n*m) lookup.

**Current**:
```elixir
allocations
|> Enum.filter(fn alloc -> alloc.pilot_id != nil end)
|> Enum.map(fn alloc ->
  pilot = Enum.find(pilots, &(&1.id == alloc.pilot_id))  # O(n) per allocation
  %{alloc | pilot: pilot}
end)
```

**Improvement**: Build lookup map first:
```elixir
pilot_map = Map.new(pilots, &{&1.id, &1})

allocations
|> Enum.filter(& &1.pilot_id)
|> Enum.map(fn alloc ->
  %{alloc | pilot: Map.get(pilot_map, alloc.pilot_id)}
end)
```

defp extract_prefixed_param(params, prefix) do
  Enum.find_value(params, fn
    {key, value} when is_binary(key) ->
      case String.split(key, prefix) do
        ["", id_string] -> {String.to_integer(id_string), value}
        _ -> nil
      end
    _ -> nil
  end)
