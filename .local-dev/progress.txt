# Reminders

- Always add adequate test coverage for new code you write
- This should include tests for Phoenix LiveViews, to assert that forms are working as expected
- When adding significant amounts of code, invoke the elixir-expert agent to check over it
- If you have any feedback or suggestions for subsequent runs of this script, add them to 'Learnings' in this file. Aim to improve this process by noting any significant causes of failure or things that could be done better.
- Important: Update this file to indicate complete items, when you complete them

# Learnings
- When adding Phoenix LiveViews, agents have a tendency to implement ad-hoc methods to modify state in the DB. It's better to think of these as being an entire form, and committing to using a Changeset for validating user input.
- For complex migrations (like JSON to table), write a detailed spec first (see specs/PILOT_ALLOCATIONS.md). This captures context and decisions for future implementation.
